# LiveBoard 全站视觉与体验改进清单

日期：2026-07-19

本清单来自对本地运行版本的全站视觉巡检，覆盖公开首页、登录页、AI、文档、课件、文件、练习、论坛、个人主页与设置，以及管理中心全部页面。实现时以当前产品决策为准：保持扁平、高密度、少卡片的工作台风格，不把 `/app` 改造成统计仪表盘，不恢复已删除的页面级大标题和营销式副标题。

## P0：窄屏与基础可用性

- [x] 小于等于 760px 时取消固定 `116px + content` 应用网格，导航改为不占用正文宽度的移动端栏。
- [x] 文档页窄屏不再同时挤压文件夹树和内容表格，提供清晰的“位置 / 当前目录”切换。
- [x] 课件编辑器窄屏回落为单列，不产生横向滚动或竖排文字。
- [x] 文档编辑器窄屏按“编辑 → 预览”顺序重排，格式工具栏可换行且不溢出。
- [x] 练习作答、批改、管理表格在窄屏保持操作可达，不被固定操作条遮挡。
- [x] 390px 与 320px 下完成重排验证；除确需二维布局的表格和幻灯片外，不要求双向滚动。

## P1：全站信息层级与一致性

- [x] 统一用户可见名称为“练习”，删除创建流程中的“测验名称”“创建测验”等混用。
- [x] 详情页明确显示当前对象名称、状态和必要操作，不使用只有“题目”“开放”等弱上下文标题。
- [x] 排序控件显示当前规则，不只显示无文字图标。
- [x] 列表、空状态与加载状态使用稳定的版式，减少无意义的大面积空白。
- [x] 危险操作收进更多菜单或独立危险区，避免与日常操作并列。
- [x] 长文、AI 回答和文档预览使用合理的阅读行宽。
- [x] 所有长表单统一显示保存中、已保存时间、保存失败与离开保护。

## AI

- [x] 限制回答正文宽度，强化用户问题与回答的对应关系。
- [x] 参考资料继续保持紧凑链接；来源过多时合并为“另有 N 个来源”。
- [x] 历史记录提供清晰的置顶、重命名、删除和搜索结果状态。
- [x] 增加停止生成、重新生成和继续回答入口。
- [x] 缩短短问题场景下的输入区高度。

## 文档

- [x] 当前目录、面包屑、排序和新建操作形成清晰工具栏。
- [x] 长文件夹名称提供完整名称提示，窄屏不压缩主内容区。
- [x] 阅读页保持 760–860px 可读行宽并提供文档目录/标题锚点。
- [x] Markdown 不支持项通过导入报告呈现，避免技术性降级文本持续污染正文。
- [x] 编辑器使用上下文格式工具，减少每个区块重复工具造成的噪音。
- [x] 编辑器提供大纲和区块快速跳转。

## 课件

- [x] 列表稳定展示名称、创建者、更新时间和页数/内容数量等可辨识信息。
- [x] 编辑器支持拖拽排序，并保留无障碍的上移/下移操作。
- [x] 文档段落与嵌套练习使用明确的标签切换。
- [x] 图片支持适应、填充和原始比例三种展示方式。
- [x] 展示页提供缩略图跳页、键盘帮助和演讲者辅助信息。

## 练习

- [x] 创建器支持题目复制、拖拽排序、预览和发布前完整性检查。
- [x] 创建器支持草稿自动保存和恢复。
- [x] 题目支持必答设置、题库复用和文本批量导入。
- [x] 作答页显示练习名称、答题进度与题目导航。
- [x] 提交后明确区分当前作答和历史提交，防止误认为仍可再次提交。
- [x] 批改页支持上一份/下一份、键盘切换和未完成评分提示。

## 论坛

- [x] 列表强化标题、作者、版块、状态和回复时间的层级。
- [x] 回复时明确显示回复对象并支持草稿自动保存。
- [x] 嵌套回复改善层级追踪，低频操作按需显示。
- [x] 增加未读、提及、关注主题和帖子内搜索能力。
- [x] 论坛内容可紧凑关联相关文档、课件或练习。

## 文件

- [x] 未选择文件时折叠详情栏，释放主列表宽度。
- [x] 列表视图提供完整文件名、上传者、时间、容量和引用数。
- [x] 引用数可查看具体引用来源。
- [x] 删除收进更多菜单，被引用资源显示不可删除原因。
- [x] 文件页只展示当前用户上传的资源，并支持批量选择与批量下载。

## 个人主页与设置

- [x] 个人主页在权限允许时展示用户公开创建的课件和论坛主题。
- [x] 设置页统一 Banner、头像、资料和密码区的操作密度。
- [x] 资料保存提供明确的保存状态，窄屏操作保持可达。

## 管理中心

- [x] 总览改为真正的待处理事项，不重复顶部导航入口。
- [x] 成员支持搜索、角色/状态过滤与批量操作。
- [x] 容量支持按使用率排序和接近上限提醒。
- [x] 权限组支持批量添加/移除成员。
- [x] 文档默认权限显示未保存/已保存状态。
- [x] AI 设置区分全局启停、限额和模型配置的风险层级。
- [x] 系统时区的常用项显示明确可点击状态。

## 全站能力与验证

- [x] 建立统一活动通知入口，承载练习发布、批改、文档更新与论坛回复。
- [x] 所有主要页面覆盖空、少量、大量、加载失败和无权限状态。
- [x] 补充桌面与窄屏视觉回归测试，防止 CSS 级联重新破坏布局。
- [x] 运行 `pnpm format:check`、`pnpm typecheck`、`pnpm test`、`pnpm build` 或 `pnpm validate`。
- [x] 使用真实本地服务逐页复查桌面和窄屏，并记录剩余限制。

## 实施与验证记录

- 2026-07-19：上述改进已完成，包含前端交互、响应式布局、API、共享类型、Prisma schema 与 migration；本地 migration 已成功应用。
- 桌面端以 1280px 宽度逐页复查公开首页、登录页、全部应用主页面、动态详情/编辑/展示页，以及管理中心全部子页面，未发现页面级横向溢出。
- 窄屏以 390px 与 320px 复查 AI、文档、课件、文件、练习、论坛、个人主页、管理中心等关键页面；移动导航、文档双栏切换和个人主页单列布局均完成实机浏览器校验。
- 活动通知入口已使用真实 API 数据验证未读数量、列表内容与一键已读行为；论坛、个人公开内容和文件引用等新增能力已通过页面与接口复查。
- 新增响应式 CSS 回归测试，并保留现有组件对空状态、错误和权限状态的测试覆盖；真实数据同时覆盖空列表、少量内容与长列表场景。
- `pnpm validate` 已通过：格式检查、类型检查、共享包 9 项测试、API 19 个测试套件共 106 项、Web 24 个测试文件共 98 项，以及 API/Web 生产构建均成功。
- 剩余本地数据限制：文件库中一个历史图片对象在 MinIO 中已不存在，因此其缩略图仍显示加载失败；这是已有存储数据缺失，不是本次布局或组件回归。
